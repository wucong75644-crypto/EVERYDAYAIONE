> **日期**：2026-03-10

## 一、紧急修复：路由提示词 Bug

### 根因分析

提示词自相矛盾 + 缺少"不确定就问"的兜底策略：

```
工具描述：  "生成/画/绘制/创作/修改图片时调用"     ← 包含"修改"
重要规则：  "仅当用户明确要求「生成/画/制作」时才调用"  ← 不包含"修改"
```

大脑（千问）遵守了更严格的"重要"段，把"修正图片"当成非生成请求，直接走了 text_chat。

## 二、第 1.5 步：自主进化闭环 — 信号接入 + 意图学习

> 现有知识库基础设施只接了"模型表现"一条数据线，大量高价值信号被丢弃。
> 接入这些信号后，系统能从用户行为中自主学习，越用越聪明。

### 现状诊断：信号捕获率

| 类别 | 当前状态 | 捕获率 | 优先级 |
|------|---------|--------|--------|
| 意图路由决策 | 只写日志 | 0% | 🔴 最高 |
| 用户反馈（retry/regenerate） | 完全没记录 | 0% | 🔴 最高 |
| 记忆检索效果 | 只写日志 | 40% | 🟠 高 |
| 生成参数效果 | 存了参数没分析 | 20% | 🟠 高 |
| 搜索结果质量 | 只记了 query | 10% | 🟡 中 |
| 错误模式分类 | 记了 error_code | 60% | 🟡 中 |
| Agent 循环效率 | 日志级别 | 30% | 🟢 低 |

### 阶段 A：信号接入（接水管）

把已产生但未捕获的信号接入 `knowledge_metrics`：

| 信号 | 接入位置 | 记录内容 | 价值 |
|------|---------|---------|------|
| **意图路由决策** | `agent_loop.py` / `intent_router.py` 路由完成后调 `record_metric` | tool_name、recommended_model、routed_by | 知道哪些意图被误判 |
| **用户反馈** | `message_generation_helpers.py` 处理 retry/regenerate 时调 `record_metric` | feedback_type（retry/regenerate/regenerate_single）、原 model_id | 最直接的质量信号 |
| **隐式正反馈** | 定时统计未被 retry 的消息 | 无操作 = 满意 | 正样本基线 |
| **记忆检索效果** | `memory_service.py` | 检索条数 → 过滤后条数 → 注入条数 | 优化记忆阈值 |
| **搜索结果质量** | `intent_router.py` / `agent_loop.py` | 搜索耗时、结果长度、是否被引用 | 优化搜索模型选择 |
| **生成参数效果** | 将 aspect_ratio/resolution/prompt_length 与 retry 率关联 | 参数组合 + 是否被 retry | 知道哪些参数组合效果好 |

**可能需要的 DB 变更**：`knowledge_metrics` 表新增 `routing_tool`、`feedback_type` 等字段

### 阶段 B：意图学习闭环（让系统学会新表达）

大脑不确定 → **给选项引导用户** → 记录选择结果 → 写入知识库 → 下次直接路由

```
用户："修正图片"
    ↓
大脑不确定 → ask_user（带选项引导）：
    "我注意到你上传了一张图片，你想要：
     1. 编辑这张图片（裁剪/调整/风格变换）
     2. 基于这张图片生成新图片
     3. 用这张图片生成视频
     4. 只是想聊聊这张图片"
    ↓
用户选择：1（编辑图片）
    ↓
记录：{
  原始表达: "修正图片",
  用户选择: "编辑图片",           ← 结构化，不需要再解析
  确认意图: "generate_image",
  选项索引: 1
}
    ↓
写入 knowledge_nodes（category="intent_pattern", source="user_confirmed"）
    ↓
下次任何用户说"修正图片" → 大脑从知识库读到此模式 → 直接路由
```

**选项引导的好处**：
- 用户点一下就完成确认，体验好
- 选项本身就是意图分类，确认结果天然是结构化数据
- 写入知识库更干净，不需要 NLP 解析用户的自由文本回答

**实现要点**：
- `agent_loop.py`：ask_user 确认后，下一轮路由成功时，自动提取（原始表达 → 最终工具）写入知识库
- 新增 category="intent_pattern"，subcategory=tool_name
- 知识注入时自动检索相关意图模式

### 阶段 C：定期提炼（从个案到通用规则）

定时任务用大模型分析意图模式，归纳通用规则，全用户共享：

```
原始数据（100条）：
  "修正图片" → generate_image（用户选了"编辑图片"）
  "矫正图片" → generate_image（用户选了"编辑图片"）
  "调整图片大小" → generate_image（用户选了"编辑图片"）
    ↓
大模型提炼：
  "修正/矫正/纠正/调整 + 图片/照片 → 图片编辑意图"
    ↓
写入 knowledge_nodes（source="distilled", confidence=0.9）
    ↓
所有用户共享
```

**实现要点**：
- `background_task_worker.py` 新增定时任务（每日/每周）
- 从 intent_pattern 类知识中聚合 → 调大模型归纳 → 写回知识库
- 与现有 `model_scorer.py` 并行，复用 `add_knowledge` 去重机制
