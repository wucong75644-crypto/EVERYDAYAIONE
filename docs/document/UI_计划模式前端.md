# UI: 计划模式前端交互

> 版本：v1.0 | 日期：2026-04-25 | 状态：待实施
> 前置：后端计划模式 v3 已实施（TECH_Agent计划模式v3.md）

## 1. 需求

让用户可以手动强制进入计划模式（类似 Claude Code 的 Shift+Tab）。
当前自动模式已能工作（LLM 自动判断），手动模式是补充。

## 2. UI 设计（来自 v2.0 §5）

### 2.1 模式选择器

在输入框右下角"按使用量计费"位置，替换为模式选择器：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  发送消息...                                                    │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│ ✦ 智能 ∨  ⚙  ⊛ 深度思考  📁 工作区          [自动模式 ▾] 📎 🎤│
└─────────────────────────────────────────────────────────────────┘

点击 [自动模式 ▾] 弹出选项：
  ┌──────────────┐
  │ ● 自动模式    │  ← 默认，AI 自行判断
  │ ○ 计划模式    │  ← 强制先规划再执行
  └──────────────┘
```

### 2.3 UI 细节

| 元素 | 自动模式（默认） | 计划模式 |
|------|----------------|---------|
| 输入框边框 | 默认样式 | 绿色细边框（`var(--success)` 低透明度） |
| 右下角文字 | "自动模式 ▾" | "计划模式 ▾"（文字变绿色） |
| 发送行为 | AI 自行判断 | 系统提示词注入强制计划模式 |
| 模式持续 | - | 单次对话有效，刷新恢复自动模式 |
| 色系 | 与"深度思考"蓝色胶囊同一设计语言 |

## 3. 技术方案

### 3.1 前端

**状态管理**：在 chatStore 中加 `planMode: boolean`（默认 false，刷新重置）

**组件**：输入框区域新增 PlanModeSelector 组件
- 位置：替换"按使用量计费"文字
- 交互：点击弹出 Popover，选择模式
- 视觉：选中计划模式时输入框加绿色边框

**WebSocket 消息**：发送消息时附带标记
```json
{
  "type": "send_message",
  "content": "查供应商纸制品01的商品，用编码查订单",
  "plan_mode": true
}
```

### 3.2 后端

**chat_handler.py**：从 WebSocket 消息中读取 `plan_mode`

**系统提示词注入**：`plan_mode=true` 时，在 TOOL_SYSTEM_PROMPT 后追加：
```
=== 计划模式已激活（用户手动开启）===
无论查询复杂还是简单，都必须进入计划模式流程：
1. 调 erp_analyze 分析任务结构
2. 展示执行方案
3. 等用户确认后再执行
此规则覆盖"直接模式"判断。
```

### 3.3 改动文件

| 文件 | 改动 |
|------|------|
| 前端 chatStore | 加 `planMode` 状态 |
| 前端 ChatInput 组件 | 加 PlanModeSelector |
| 前端 WebSocket 发送 | 附带 `plan_mode` 参数 |
| 后端 chat_handler.py | 读取 `plan_mode` 参数 |
| 后端 chat_tools.py 或 chat_handler.py | 条件注入计划模式提示词 |
