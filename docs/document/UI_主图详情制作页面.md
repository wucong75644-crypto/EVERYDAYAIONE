> 版本：v1.0
> 日期：2026-07-11
> 状态：UI 方案已确认
> 任务等级：A级
> 关联方案：`TECH_电商图片Agent_v3.md`

## 一、设计目标与范围

### 1.1 目标

为登录用户提供独立的“主图&详情图”制作页面。用户从 Chat 左侧栏进入，通过“输入 → 分析中 → 确认规划 → 生成中 → 完成”五步流程完成一组电商图片制作。

页面视觉以流影 AI `studio-genesis` 导出页面、HAR 和截图为参考，业务行为使用 EVERYDAYAIONE 自有能力，不复制第三方代码或接口。

### 1.2 MVP 范围

- 支持主图和详情图。
- 支持上传产品图、填写要求和选择生成参数。
- 支持 AI 帮写需求。
- 支持生成前确认和编辑图片规划。
- 支持逐张展示生成进度与结果。
- 支持单张下载、失败重试和再次制作。

### 1.3 暂不包含

- 广告图真实生成能力（MVP 完全隐藏入口）。
- 历史项目列表。
- 打包下载。
- 二次精修。
- 多模型品牌档位和风格策略库。
- 移动端完整制作能力。

## 三、页面结构

### 3.1 页面信息

- 页面名称：主图&详情图制作。
- 路由：`/detail-page`。
- 权限：登录用户。
- 桌面端目标宽度：优先适配 1280px 及以上。
- 页面性质：独立页面，不嵌入 Chat 内容区。
- 独立页面不保留 Chat 左侧栏；顶部保留“返回聊天”、剩余积分和用户入口。

### 3.2 总体布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 每日AI  主图&详情图                     返回聊天  积分  用户 │
├──────────────────────────────────────────────────────────────┤
│                 AI 帮写需求，一键生成详情图组                │
│      上传产品图，AI 分析并规划多角度、多场景电商图片         │
│                                                              │
│  ① 输入 ── ② 分析中 ── ③ 确认规划 ── ④ 生成中 ── ⑤ 完成  │
├──────────────────────┬───────────────────────────────────────┤
│ 左侧输入/设置区       │ 右侧规划、进度和结果工作区            │
│                      │                                       │
│ 上传、类型、平台      │ 随当前步骤切换内容                    │
│ 要求、语言、比例      │                                       │
│ 清晰度、数量          │                                       │
├──────────────────────┴───────────────────────────────────────┤
│ 页面级错误或任务状态提示                                    │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 顶部导航

- 左侧显示“每日AI”和当前功能名。
- 右侧提供“返回聊天”、剩余积分和用户入口。
- “返回聊天”默认回到用户离开前的 Chat 地址；无历史地址时回到 `/chat`。
- 任务生成中点击返回时，明确提示“任务会继续执行，可稍后返回查看”。

### 3.4 五步进度条

- 始终显示五个固定步骤及当前位置。
- 已完成步骤：强调色对勾与已完成文字。
- 当前步骤：强调色背景、清晰的步骤名称。
- 未开始步骤：中性边框和灰色文字。
- 不允许直接点击未来步骤。
- Step 3 之后允许返回已完成步骤，但必须提示可能使现有规划失效。

## 四、五步视图设计

### 4.1 Step 1：输入

#### 左侧输入区

1. 产品图上传
   - 上传区只分为“产品图”和“参考图”两个板块。
   - 产品图为必填，至少上传 1 张；未上传产品图时禁用“分析产品”。
   - 参考图为可选，用于提供氛围、构图、风格或细节参考。
   - 两个板块共用总额度，合计最多 9 张；不分别限制各板块数量。
   - 系统不从参考图中自动指定产品图。
   - 支持 JPG、PNG、WebP。
   - 上传后显示缩略图，可预览、删除和继续添加。
   - 单张失败不清空其他已上传图片。
   - 明确提示建议上传正面、侧面、背面和细节图。
2. 生成类型
   - `主图`、`详情图` 两个可用选项。
   - MVP 不显示不可用的广告图入口。
3. 目标平台
   - 默认“智能匹配”。
   - 首期提供淘宝、天猫、京东、拼多多。
4. 图片要求
   - 多行输入框。
   - 提示用户填写产品名称、卖点、目标人群、平台和图片风格。
   - 右下角提供“AI 帮写”。
5. 目标语言
   - 中文简体、无文字，默认中文简体。
6. 生成参数
   - 模型：首期隐藏模型选择，由系统自动使用默认模型。
   - 尺寸比例：主图默认 1:1，详情图默认 3:4。
   - 清晰度：1K、2K、4K。
   - 数量：通过下拉框选择 1–9 张，默认 1 张。
7. 主操作
   - 按钮文案：`分析产品`。
- 未上传任何产品图时禁用，并说明“请至少上传 1 张产品图”。

#### 右侧初始状态

- 居中显示图片规划图标。
- 主文案：`上传产品图并填写要求后，点击“分析产品”开始`。
- 不展示空白大区域之外的无关功能。

### 4.2 Step 2：分析中

- 左侧保留用户输入，但整体锁定，避免分析过程中参数变化。
- 右侧显示当前分析阶段，不只使用无限旋转动画。
- 推荐阶段文案：
  1. 正在识别产品主体。
  2. 正在分析材质与结构。
  3. 正在提取核心卖点。
  4. 正在规划图片组合。
- 提供“取消分析”。
- 取消后返回 Step 1，并保留全部输入内容。
- 超时或失败后保留上传内容和表单，提供“重新分析”。
- 分析成功后自动进入 Step 3，不增加额外确认操作。
- 没有真实子阶段进度时只显示阶段文案，不伪造百分比。
- 动态进度使用可被辅助技术感知的状态文本。

### 4.3 Step 3：确认规划

- 右侧以列表或网格展示方案卡片。
- 每张卡片包含：
  - 序号和图片角色。
  - 构图与使用目的。
  - 标题和副标题。
  - 图片比例。
  - 是否包含文字。
  - 可折叠的高级提示词。
- 单卡操作：编辑、删除。
- 标题、副标题和高级提示词均允许编辑。
- 页面操作：返回修改需求、重新规划、确认生成。
- 确认按钮显示实际数量和预计积分，例如 `确认生成 5 张 · 预计 125 积分`。
- 删除后至少保留 1 张；最后一张不可删除，只能重新规划。
- 重新规划前二次确认，避免覆盖用户已经编辑的内容。
- 积分不足时停留在 Step 3，明确显示现有积分与所需积分。

### 4.4 Step 4：生成中

- 顶部显示整组进度，例如 `正在生成 3/5`。
- 每张图片独立显示以下状态：
  - 等待中。
  - 生成中。
  - 已完成。
  - 生成失败。
- 已完成图片立即展示，不等待整组完成。
- 已完成图片允许在整组生成过程中提前查看和下载。
- 单张失败显示原因、退款状态和“重试该张”。
- 单张失败不阻塞其他图片。
- Step 4 不提供整组取消；已提交的任务继续执行并按实际结果结算。
- 用户离开页面时明确提示任务仍会继续。
- 返回页面后以服务端真实任务状态恢复，不显示虚假进度。

### 4.5 Step 5：完成

- 展示整组结果网格。
- 每张图片显示角色、比例和状态。
- 单张操作：查看大图、下载、重新生成。
- 页面操作：再次制作、返回修改方案。
- “再次制作”返回 Step 1，保留产品图、参考图和全部生成设置，仅清空上一轮结果。
- 单张重新生成前显示预计积分并二次确认；新结果不覆盖旧图，由用户选择保留版本。
- 部分失败时仍可进入完成页，顶部明确显示成功数和失败数。

## 五、状态与异常设计

| 状态 | 触发条件 | 显示内容 | 用户操作 |
|---|---|---|---|
| 初始 | 首次进入 | 输入表单和右侧引导 | 上传并填写 |
| 上传中 | 图片上传 | 单图进度与禁用提交 | 可取消单图 |
| 表单错误 | 缺少必填或格式错误 | 对应字段下方文字错误 | 修正后提交 |
| AI 帮写中 | 请求进行中 | 按钮 Loading，输入框保留 | 不重复提交 |
| 分析中 | 产品分析进行中 | 分阶段进度 | 可取消 |
| 分析失败 | 超时或服务错误 | 明确原因和重试按钮 | 重新分析 |
| 规划就绪 | 分析成功 | 可编辑方案卡片 | 确认生成 |
| 生成中 | 已确认生成 | 整体与单张进度 | 可离开页面 |
| 单张失败 | 某张失败 | 卡片内错误、退款和重试 | 重试该张 |
| 积分不足 | 生成前校验失败 | 所需积分与当前积分 | 返回调整数量 |
| 部分成功 | 存在成功和失败 | 成功结果与失败卡并存 | 下载或重试 |
| 全部完成 | 所有图片完成 | 结果网格 | 下载或再次制作 |
| 恢复中 | 刷新后检测到任务 | 恢复提示和骨架 | 等待恢复 |

错误信息必须说明“哪里错、为什么、如何修复”，不能只显示错误码或红色边框。Toast 仅用于轻量成功提示，关键失败必须在对应区域持续展示。

积分采用按张结算：成功图片正常扣费，失败图片按张退款。即使失败数量达到总数的一半或以上，也不退还成功图片的积分；成功结果继续保留，并提供“仅重试失败图片”。

## 六、响应式设计

### 6.1 桌面端（≥1280px）

- 使用左侧固定设置区和右侧自适应工作区。
- 五步进度条完整横向展示。
- 结果区域根据比例自动使用 2–4 列网格。

### 6.2 小屏桌面/平板（768–1279px）

- 顶部导航保留核心操作。
- 输入区与工作区改为上下布局。
- 五步进度条允许横向滚动，不压缩步骤文字。

### 6.3 手机端（<768px）

- MVP 提供可操作的单列布局。
- 上传、填写要求、分析、查看进度和下载可用。
- 复杂方案编辑提示用户前往桌面端完成。

## 八、交互与可访问性底线

- 所有表单控件都有可见 Label。
- 键盘可以访问所有按钮、下拉框和方案卡操作。
- Focus 状态清晰，不能被自定义样式移除。
- 表单错误与字段建立明确关联，并将焦点移动到首个错误字段。
- 加载、成功、失败和进度变化有文本反馈。
- 图标按钮提供可访问名称和 Tooltip。
- 图片缩略图提供产品图序号或用途说明。
- 禁用按钮同时说明不可用原因。
- 自动动画不影响阅读，持续动画需尊重减少动态效果设置。

## 九、第一阶段 UI 开发边界

第一阶段只实现可点击的 UI 骨架和 Mock 状态：

1. Chat 左侧栏入口。
2. `/detail-page` 独立受保护路由。
3. 顶部导航和五步进度条。
4. Step 1 完整输入表单。
5. Step 2–5 使用 Mock 数据展示完整状态。
6. 桌面端、小屏桌面/平板和手机端布局。
7. UI 单元测试和浏览器视觉验收。

第一阶段不连接真实分析、生图、积分扣除和任务恢复接口。真实业务链路在后续技术设计确认后分阶段接入。

## 十、UI 验收标准

- 从 Chat 左侧栏“AI 记忆”上方可进入独立页面。
- 页面视觉与参考截图保持一致的布局层次，同时使用每日AI主题 Token。
- 五步状态可通过 Mock 数据完整切换，无卡死或布局跳动。
- 主图和详情图切换会更新默认比例和字段文案。
- 所有下拉项、禁用态、错误态、空态、加载态和完成态均可演示。
- 1280px、1440px 和 1920px 宽度下无遮挡和横向溢出。
- 键盘可完成 Step 1 表单操作，状态变化具有文本反馈。
- 不出现第三方品牌、接口地址、Token 或第三方业务代码。
