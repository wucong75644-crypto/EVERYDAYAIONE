# REQ: 模型广场首页改版

> **版本**: v1.1 | **状态**: 需求评审 | **日期**: 2026-03-09

---

## 一、需求概述

将独立的"模型广场"页面整合到首页，首页改版为**品牌入口 + 模型广场**。用户在首页即可浏览所有模型、订阅模型、查看详情，订阅后可跳转聊天页使用。

### 核心目标
1. 首页从空白落地页 → 模型广场（提升首屏价值）
2. 订阅制：用户需订阅模型后才能使用（为未来收费预留）
3. 聊天跳转：已订阅模型可从详情面板跳转聊天页，未订阅则需先订阅

---

## 二、用户角色与权限

| 角色 | 可见内容 | 可操作 |
|------|---------|--------|
| 未登录用户 | 模型卡片（名称+描述+能力+费用） | 无按钮，顶部CTA引导注册 |
| 已登录用户 | 全部内容 | 订阅/取消、查看详情、跳转聊天（需已订阅） |

### 3.2 分类Tab说明

| Tab | 行为 |
|-----|------|
| 全部 | 按类别分组显示所有模型（聊天→图片→视频），每组有标题分隔 |
| 💬聊天 | 只显示聊天模型，平铺卡片网格 |
| 🎨图片 | 只显示图片模型 |
| 🎬视频 | 只显示视频模型 |

- 默认选中"全部"
- Tab 切换不刷新页面，纯前端过滤
- 搜索框在所有 Tab 下都生效

### 3.3 搜索功能

- 实时过滤（输入即搜索，无需点击按钮）
- 搜索范围：模型名称 + 模型描述
- 空结果显示："未找到匹配的模型"
- 清空搜索恢复全部显示

## 四、模型卡片设计

### 4.1 卡片内容

```
┌─────────────────────┐
│ [标签]        [图标] │  ← 标签：🟢免费 / ⭐热门 / 🔥新上线 / 维护中
│                     │
│  模型名称            │  ← 如 "GPT-5.4"
│  一句话描述          │  ← 如 "OpenAI 最新旗舰 | 百万上下文"
│                     │
│  [能力图标] [能力图标]│  ← 如 📝文本 🖼️图片 🎤音频 📄PDF 🔧工具
│                     │
│  X 积分/次           │  ← 费用信息
│                     │
│  [订阅按钮]          │  ← 已登录显示；未登录不显示
└─────────────────────┘
```

### 4.2 能力图标映射

| 能力 | 图标 | 条件 |
|------|------|------|
| 文本对话 | 📝 | type === 'chat' |
| 图片理解 | 🖼️ | capabilities.vqa === true |
| 音频输入 | 🎤 | capabilities.audioInput === true |
| PDF理解 | 📄 | capabilities.pdfInput === true |
| 工具调用 | 🔧 | capabilities.functionCalling === true |
| 文生图 | 🎨 | capabilities.textToImage === true |
| 图像编辑 | ✏️ | capabilities.imageEditing === true |
| 文生视频 | 🎬 | capabilities.textToVideo === true |
| 图生视频 | 🎞️ | capabilities.imageToVideo === true |

### 4.3 卡片标签逻辑

| 标签 | 条件 | 样式 |
|------|------|------|
| 🟢 免费 | credits === 0 | 绿色背景 |
| ⭐ 热门 | 手动标记（后续从后端获取） | 金色背景 |
| 🔥 新上线 | 上线 < 7天（后续） | 红色背景 |
| 维护中 | status === 'maintenance' | 橙色背景 |
| 即将上线 | status === 'coming_soon' | 灰色背景 |

> V1 阶段：免费标签根据 credits === 0 自动显示，其他标签暂不实现

### 4.4 卡片状态与订阅按钮

| 状态 | 按钮文案 | 按钮样式 | 点击行为 |
|------|---------|---------|---------|
| 未订阅 | 订阅 | 蓝色实心 | 调用订阅API |
| 已订阅 | ✓ 已订阅 | 灰色边框 | 无操作（在详情中可取消） |
| 默认模型 | ✓ 默认 | 灰色边框+禁用 | 不可取消 |
| 维护中 | 维护中 | 灰色禁用 | 不可点击 |
| 未登录 | 不显示按钮 | — | — |

### 4.5 卡片点击交互

- **点击卡片整体**（非按钮区域）→ 弹出模型详情面板
- **点击订阅按钮** → 直接订阅/提示登录（阻止冒泡）

## 五、模型详情面板

### 5.1 布局

```
┌──────────────────────────────────────────────────────┐
│  GPT-5.4                                      [×关闭] │
├──────────────────────────────────────────────────────┤
│                                                      │
│  OpenAI 最新旗舰 | 百万上下文 | 全能                  │
│                                                      │
│  ── 模型能力 ──                                       │
│  [📝 文本对话] [🖼️ 图片理解] [🔧 工具调用]            │
│  [📊 JSON输出] [⚡ 流式响应]                           │
│                                                      │
│  ── 规格参数 ──                                       │
│  上下文长度：1,050,000 tokens                         │
│  最大图片数：10 张                                    │
│  图片大小限制：≤20MB                                  │
│                                                      │
│  ── 费用 ──                                           │
│  单次消耗：5 积分                                     │
│  (图片模型显示分辨率价格表)                            │
│  (视频模型显示时长价格表)                              │
│                                                      │
│  ── 操作 ──                                           │
│                                                      │
│  [↗ 前往聊天页]  ← 已订阅：跳转 /chat?model=xxx       │
│                  ← 未订阅：显示"请先订阅" + [订阅]按钮  │
│                                                      │
│  [取消订阅]  ← 仅已订阅非默认模型显示                  │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 5.2 详情面板交互

- **触发**：点击模型卡片（非按钮区域）
- **展现形式**：右侧滑入面板（Drawer），背景半透明遮罩
- **关闭方式**：点击 × / 点击遮罩 / 按 ESC
- **未登录用户**：不显示"前往聊天页"按钮，显示"登录后使用"提示 + [立即注册] 按钮
- **已登录 + 已订阅**：显示 [↗ 前往聊天页] 按钮，点击跳转 `/chat?model=xxx`
- **已登录 + 未订阅**：显示"请先订阅才能使用" + [订阅] 按钮，订阅成功后按钮变为 [↗ 前往聊天页]

### 5.3 费用展示逻辑

| 模型类型 | 费用展示 |
|---------|---------|
| 聊天模型 (credits: number) | `X 积分/次` 或 `免费` |
| 图片模型 (credits: Record) | 分辨率价格表：`1K: 18积分 / 2K: 18积分 / 4K: 24积分` |
| 视频模型 (videoPricing) | 时长价格表：`10秒: 30积分 / 15秒: 45积分` |

## 六、订阅系统

### 7.1 现有基础

| 组件 | 状态 |
|------|------|
| `user_subscriptions` 表 | ✅ 已存在（user_id, model_id） |
| `models` 表 | ✅ 已存在（id, is_default） |
| 注册自动订阅默认模型 | ✅ 已实现（auth_service._subscribe_default_models） |
| 前端订阅API | ❌ 不存在 |
| 后端订阅管理路由 | ❌ 不存在 |
| 聊天页模型过滤（只显示已订阅） | ❌ 不存在（当前显示所有模型） |

### 7.2 需要新增的 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET /api/models` | GET | 获取所有模型列表（含 is_default、status） |
| `GET /api/subscriptions` | GET | 获取当前用户已订阅的模型ID列表 |
| `POST /api/subscriptions/{model_id}` | POST | 订阅模型 |
| `DELETE /api/subscriptions/{model_id}` | DELETE | 取消订阅（默认模型不可取消） |

### 7.3 订阅与聊天页联动

- 聊天页模型选择器：**只显示已订阅的模型**
- 智能模式(auto)：从已订阅模型中选择最优模型
- 用户新订阅模型后，聊天页模型列表实时更新

### 7.4 默认模型（不可取消订阅）

V1 阶段默认模型（is_default = true）：
- `gemini-3-flash` — 免费聊天入门
- `gemini-3-pro` — 免费高级聊天

> 具体默认模型列表后续可在管理后台配置
