# Everything Claude Code 使用指南

> 本目录包含完整的 everything-claude-code 参考资料，可按需手动调用。

## 目录结构

```
everything-claude-code-reference/
├── agents/          # 12 个专业代理
├── skills/          # 22 个技能模块
├── rules/           # 8 个规则文件（已有 2 个在自动加载）
├── commands/        # 命令模板
├── contexts/        # 上下文模板
└── docs/            # 文档
```

---

## 一、Agents（代理）调用方法

### 调用语法
```
读取 ~/.claude/plugins/marketplaces/everything-claude-code-reference/agents/[代理名].md 并按照指南执行
```

### 代理列表

| 代理 | 文件 | 用途 | 使用场景 |
|------|------|------|---------|
| **architect** | architect.md | 架构设计 | 系统设计、技术选型、扩展性规划 |
| **planner** | planner.md | 任务规划 | 功能拆分、实现计划、依赖分析 |
| **code-reviewer** | code-reviewer.md | 代码审查 | 代码质量、安全检查、性能分析 |
| **security-reviewer** | security-reviewer.md | 安全审查 | OWASP Top 10、密钥检测、漏洞扫描 |
| **tdd-guide** | tdd-guide.md | TDD 指导 | 测试驱动开发、覆盖率要求 |
| **build-error-resolver** | build-error-resolver.md | 构建修复 | TypeScript/编译错误快速修复 |
| **refactor-cleaner** | refactor-cleaner.md | 重构清理 | 死代码检测、依赖清理 |
| **doc-updater** | doc-updater.md | 文档更新 | 自动生成文档、保持同步 |
| **e2e-runner** | e2e-runner.md | E2E 测试 | Playwright 测试、用户流程验证 |
| **database-reviewer** | database-reviewer.md | 数据库审查 | PostgreSQL 优化、RLS、索引 |
| **go-build-resolver** | go-build-resolver.md | Go 构建修复 | Go 编译错误修复 |
| **go-reviewer** | go-reviewer.md | Go 代码审查 | Go 代码质量、并发安全 |

### 使用示例

```
# 架构设计
读取 agents/architect.md，帮我设计用户认证系统的架构

# 代码审查
读取 agents/code-reviewer.md，审查我刚提交的代码

# 安全审查
读取 agents/security-reviewer.md，检查项目的安全漏洞

# 构建错误
读取 agents/build-error-resolver.md，修复当前的 TypeScript 错误
```

## 三、Rules（规则）

### 已自动加载（每次对话）
- `~/.claude/rules/git-workflow.md` - Commit 格式、PR 流程
- `~/.claude/rules/patterns.md` - API 格式、设计模式

### 备用规则（按需调用）
位于 `rules/` 目录：

| 规则 | 用途 |
|------|------|
| agents.md | 代理使用指南 |
| coding-style.md | 编码风格（不可变性、文件组织） |
| hooks.md | Claude Hooks 配置 |
| performance.md | 模型选择、上下文管理 |
| security.md | 安全检查清单 |
| testing.md | 测试要求、TDD 流程 |

### 调用示例
```
读取 rules/security.md，按照安全清单检查代码
```
