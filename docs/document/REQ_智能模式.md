# 需求文档：智能模式（AI Agent）

## 1. 背景

当前系统中，AI 对话是无状态的 — 每条消息独立处理，不带对话历史。图片/视频任务的分发由前端规则判断（`infer_generation_type`）。用户希望 AI 具备「大脑」能力：记住上下文、理解意图、自主调用工具。

## 2. 目标

将 AI 从「无状态应答机器」升级为「有记忆、有上下文、能调度工具的 Agent」。

## 3. 核心需求

### 3.1 对话上下文（Phase 1）

**问题**：同一对话中 AI 不记得前面说了什么，无法追问和延伸。

**需求**：
- 发送消息给 LLM 时，携带同一对话的最近 N 条历史消息
- 历史消息只提取文本内容，不带图片 URL 等大体积数据
- 图片任务的用户指令文本需要保留（帮助 AI 理解上下文意图）
- 历史条数可配置（默认 10-20 条）
- 不影响现有功能，失败时降级为无历史模式

### 3.2 Agent 调度 — Function Calling（Phase 2）

**问题**：任务类型分发靠前端关键词匹配，不够智能。

**需求**：
- 千问通过 Function Calling 自主决定调用哪个工具
- 工具定义：生成图片、编辑图片、生成视频（未来可扩展）
- 千问看到对话上下文后能判断：用户是「编辑上一张图」还是「发起新任务」
- 普通聊天直接回复，不调用任何工具

### 3.3 智能模式 UI（Phase 3，与 Phase 2 配套）

**问题**：需要给用户提供 Agent 模式的入口和控制权。

**需求**：
- 模型选择新增「智能模式」选项，设为默认
- 智能模式 = 千问 Agent（上下文 + Function Calling + 自动工具调度）
- 用户可切换到具体模型（如 qwen-plus、Imagen），跳过 Agent 直接执行
- 切换模式不影响已有对话历史

## 4. 架构概览

```
┌─────────────────────────────────────────┐
│            大脑（千问 LLM）              │
│                                         │
│  长期记忆 = Mem0 跨对话记忆 ✅ 已完成    │
│  短期记忆 = 对话历史注入    Phase 1      │
│  决策能力 = Function Calling Phase 2     │
└──────────────┬──────────────────────────┘
               │ 调用工具（手脚）
       ┌───────┼───────┐
       ▼       ▼       ▼
    生成图片  编辑图片  生成视频     ← 已有 handler
   (Google)  (Google)  (未来)

UI 层：智能模式（默认）/ 手动选模型    Phase 3
```

## 5. 实施顺序

| 阶段 | 依赖 | 产出 |
|------|------|------|
| Phase 1：对话历史 | 无 | AI 能记住同一对话的上下文 |
| Phase 2：Function Calling | Phase 1 | AI 能自主调用生图/编辑/生视频 |
| Phase 3：智能模式 UI | Phase 2 | 用户可切换智能/手动模式 |

## 6. 边界约束

- 历史消息只取文本，不带图片 URL（控制 token）
- 图片任务指令文本保留在历史中（AI 需要理解上下文）
- Phase 2 的 Function Calling 使用千问原生支持
- 智能模式失败时降级为当前逻辑（前端分发）
- 手动选模型时完全跳过 Agent 层
