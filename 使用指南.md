# Claude Code 使用指南

---

## 一、EVERYDAYAI 开发流程技能（斜杠命令）

按顺序使用，也可单独触发。

| 命令 | 触发条件 | 功能 |
|------|---------|------|
| `/everydayai-requirement` | 说"我想要…""加一个功能" | 需求挖掘：结构化提问，理清需求边界，输出需求清单 |
| `/everydayai-ui-design` | 说"界面设计""页面布局" | UI设计：页面结构、交互流程、状态设计、组件清单 |
| `/everydayai-tech-design` | 说"技术方案""API设计" | 技术设计：数据库表、API接口、文件结构、任务拆分 |
| `/everydayai-implementation` | 说"开始开发""写代码" | 开发执行：深度阅读代码→基线测试→逐步实现→每步单测→代码体检→AI全链路功能测试→全部通过后才喊用户验收 |
| `/everydayai-testing` | 说"有bug""报错""改一下" | 测试修复：分析问题→提出方案→确认后修复→最小改动原则 |
| `/everydayai-restart` | 说"重启服务""restart""杀掉进程" | 重启开发服务：杀死所有旧进程→释放端口(8000/3000)→重启前后端→验证服务正常 |
| `/everydayai-git-push` | 说"推送代码""push""推到git" | Git推送：检查状态→生成commit message→提交推送到远程 |
| `/everydayai-deploy` | 说"部署""deploy""上线" | 部署生产：确认代码已推送→选择部署范围(前端/后端/全量)→执行deploy.sh→验证服务 |

## 二、辅助代理（手动调用）

调用方式：`读取 ~/.claude/plugins/marketplaces/everything-claude-code-reference/agents/[文件名] 并执行`

| 代理 | 文件名 | 功能 |
|------|--------|------|
| 架构师 | architect.md | 系统设计、技术选型 |
| 规划师 | planner.md | 功能拆分、实现计划 |
| 代码审查 | code-reviewer.md | 代码质量、安全、性能分析 |
| 安全审查 | security-reviewer.md | OWASP Top 10、密钥检测 |
| TDD指导 | tdd-guide.md | 测试驱动开发 |
| 构建修复 | build-error-resolver.md | TypeScript/编译错误修复 |
| 重构清理 | refactor-cleaner.md | 死代码检测、依赖清理 |
| 文档更新 | doc-updater.md | 自动生成/同步文档 |
| E2E测试 | e2e-runner.md | Playwright 端到端测试 |
| 数据库审查 | database-reviewer.md | PostgreSQL 优化、索引 |

## 五、常用场景速查

| 做一个新功能 | `/everydayai-requirement` 开始，按流程走完5个阶段 |

| 优化数据库 | `agents/database-reviewer.md` |

| 推送代码到Git | `/everydayai-git-push` |
| 部署到生产服务器 | `/everydayai-deploy` |
