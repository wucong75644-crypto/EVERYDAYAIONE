# 部署到生产服务器

> **触发**：用户说"部署""deploy""上线""发布到服务器"

## 执行流程

### 1. 部署前检查
- 确认当前代码已提交并推送（无未提交更改）
- 如有未提交更改，提醒用户先执行 `/everydayai-git-push`

### 2. 确认部署范围
询问用户部署范围：
- **全量部署**（前端+后端）：`./deploy/deploy.sh`
- **仅前端**：`./deploy/deploy.sh -f`
- **仅后端**：`./deploy/deploy.sh -b`

### 3. 执行部署
```bash
cd /Users/wucong/EVERYDAYAIONE && ./deploy/deploy.sh [选项]
```

### 4. 输出报告
```
✅ 部署完成

| 项目 | 状态 |
|------|------|
| Git状态 | ✅ 已推送到远程 |
| 前端构建 | ✅ 构建成功 |
| 后端检查 | ✅ 语法通过 |
| 文件同步 | ✅ 已同步到服务器 |
| 服务重启 | ✅ 服务运行中 |

访问地址：https://域名
```
